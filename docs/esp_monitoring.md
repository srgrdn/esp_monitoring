# Дашборд: ESP Monitoring

Базовый оперативный обзор для синтетических метрик устройства (ESP32 + GSM + BME280), без аналитических корреляций.

## Источник данных

- Datasource: `VictoriaMetrics` (Prometheus совместимый).
- Метрики без лейблов (скрейп со стенда генератора).

## Панели и метрики

- **RSSI (dBm)** — timeseries `rssi`; порог -90 (красный). Быстрая оценка радиосигнала.
- **Temperature (°C)** — timeseries `temp`; пороги 15 (желтый) и 30 (красный). Контроль условий среды.
- **RTT (ms)** — timeseries `rtt`; пороги 200 (желтый), 400 (красный). Задержка до сервера.
- **Packet Loss (%)** — timeseries `packet_loss`; порог 5 (красный). Потери пакетов.
- **HTTP status** — stat `http_status`; текущий код ответа (200/500).
- **Network Type** — stat `network_type`; 1=GSM, 2=LTE.

## Использование

- Подходит для оперативного просмотра состояния устройства/сети.
- Пороговые линии помогают быстро заметить отклонения, которые триггерят правила в vmalert/Alertmanager.
